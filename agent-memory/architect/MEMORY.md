# Architect Memory (global)

Last curated: 2026-06-03 (seed)
Scope: cross-project design/planning/orchestration craft (the main Opus session). Read when planning or dispatching. Hand-curated by the architect; keep it short.

## Durable Learnings
- Don't assume deadlines or commercial numbers on the user's behalf — use [TBC] placeholders and let them fill in the figures.
- Plan to the user's actual capacity and cadence (solo? part-time? team?). Size sprints to real available time, not an idealized team's two weeks.
- Reserve Opus for design / contract / cross-file judgement / triage; route bounded work down to implementer-sonnet (multi-file) or implementer-haiku (single-file/mechanical).
- Forced-Opus triggers — keep the *design* in the architect session, never silently let a cheaper tier absorb: security-sensitive changes, cross-file invariant design, schema/migration risk.
- Plan → Execute → Review: review returns to the more capable tier; never self-review.
- Orchestrating skills (`deep-research`, `code-review ultra`, `dispatch`) run only in this session — sub-agents can't spawn sub-agents.
